

# Advanced Offer, Coupon & Referral System Upgrade

## Current State

- **Coupons**: Only `discount_percent` type, generated by DB triggers (`check_milestone_rewards`, `check_referral_rewards`). No redemption/application logic exists — coupons are never actually applied to orders.
- **Offers**: Full template system with 5 designs, badges, animations, timers. No rules engine or conditional logic.
- **Referrals**: Basic referrer/referee flow with settings. No abuse detection.
- **Orders**: Placed via `place_order` RPC with credit deduction. No coupon parameter accepted.

## Database Changes

### 1. Upgrade `coupons` table with new columns

```sql
ALTER TABLE coupons ADD COLUMN IF NOT EXISTS discount_type text NOT NULL DEFAULT 'percent';
-- 'percent', 'fixed', 'free_delivery', 'bogo'
ALTER TABLE coupons ADD COLUMN IF NOT EXISTS discount_value numeric NOT NULL DEFAULT 0;
-- For fixed: NPR amount. For percent: percentage (migrate existing discount_percent values here)
ALTER TABLE coupons ADD COLUMN IF NOT EXISTS max_uses integer NOT NULL DEFAULT 1;
ALTER TABLE coupons ADD COLUMN IF NOT EXISTS use_count integer NOT NULL DEFAULT 0;
ALTER TABLE coupons ADD COLUMN IF NOT EXISTS min_order_amount numeric NOT NULL DEFAULT 0;
ALTER TABLE coupons ADD COLUMN IF NOT EXISTS max_discount_amount numeric; -- cap for percentage coupons
ALTER TABLE coupons ADD COLUMN IF NOT EXISTS applicable_products text[]; -- NULL = all products
ALTER TABLE coupons ADD COLUMN IF NOT EXISTS applicable_categories text[]; -- NULL = all categories

-- Migrate existing data
UPDATE coupons SET discount_type = 'percent', discount_value = discount_percent WHERE discount_value = 0;
```

### 2. Create `coupon_rules` table (dynamic rules engine)

```sql
CREATE TABLE coupon_rules (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name text NOT NULL,
  description text,
  rule_type text NOT NULL DEFAULT 'auto_generate',
  -- 'auto_generate' (auto-creates coupons when conditions met)
  -- 'global_code' (admin-created code anyone can use)
  conditions jsonb NOT NULL DEFAULT '{}',
  -- e.g. {"min_order_amount": 500, "user_type": "new", "category": "freefire", "min_orders": 5}
  discount_type text NOT NULL DEFAULT 'percent',
  discount_value numeric NOT NULL DEFAULT 0,
  max_discount_amount numeric,
  max_uses_per_user integer DEFAULT 1,
  max_total_uses integer,
  total_used integer DEFAULT 0,
  coupon_code text, -- for global_code type
  applicable_products text[],
  applicable_categories text[],
  starts_at timestamptz,
  expires_at timestamptz,
  is_active boolean NOT NULL DEFAULT true,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);
ALTER TABLE coupon_rules ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Admins can manage coupon rules" ON coupon_rules FOR ALL USING (is_admin()) WITH CHECK (is_admin());
CREATE POLICY "Anyone can view active rules" ON coupon_rules FOR SELECT USING (is_active = true);
```

### 3. Create `promotion_analytics` table

```sql
CREATE TABLE promotion_analytics (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  coupon_id uuid REFERENCES coupons(id) ON DELETE SET NULL,
  coupon_rule_id uuid REFERENCES coupon_rules(id) ON DELETE SET NULL,
  offer_id uuid REFERENCES offers(id) ON DELETE SET NULL,
  user_id uuid NOT NULL,
  order_id uuid REFERENCES product_orders(id) ON DELETE SET NULL,
  discount_amount numeric NOT NULL DEFAULT 0,
  event_type text NOT NULL DEFAULT 'coupon_used',
  created_at timestamptz NOT NULL DEFAULT now()
);
ALTER TABLE promotion_analytics ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Admins can view analytics" ON promotion_analytics FOR ALL USING (is_admin()) WITH CHECK (is_admin());
```

### 4. Update `place_order` function

Add optional `p_coupon_code text DEFAULT NULL` parameter. When provided:
- Validate coupon exists, belongs to user (or is global), not expired, not over max_uses, min_order_amount met
- Calculate discount, deduct from price
- Mark coupon as used / increment use_count
- Log to `promotion_analytics`
- Store coupon info in order `metadata`

### 5. Add referral abuse prevention

Create a DB function `detect_referral_abuse` that checks:
- Same IP / device fingerprint (stored in profiles metadata)
- Same email domain patterns
- Referrer and referee can't share the same device fingerprint

## Security Measures

- All coupon validation happens server-side in `place_order` RPC (SECURITY DEFINER)
- Coupon codes validated atomically with `FOR UPDATE` locks to prevent race conditions
- `max_uses` enforced server-side, not client-side
- Referral abuse: DB function checks for self-referral, duplicate device fingerprints
- Global coupon codes use `max_uses_per_user` to prevent repeated use
- All inputs validated with Zod on client, re-validated in DB function
